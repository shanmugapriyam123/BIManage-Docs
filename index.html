Installation Steps

  1. Download and run ZeManageSetup.exe from the distribution provided by your administrator.
  2. Review and accept the Privacy Policy.
  3. Enter your license key (provided by your company or project administrator).
  4. Select the Revit versions you want to install for (multi-version supported).
  5. Complete the installation wizard and restart Revit.

License Activation & Device Registration

On first launch after installation, ZeManage will prompt you to register your device. This links your machine to your organization's license. Each license key supports a configurable number of devices.

If you need to transfer your license to a new machine, contact your Company Administrator or reach out to support@zestinetech.com.

Quick Start Guide

Get up and running with ZeManage in minutes. Follow this first-time setup flow:

  1. Open Revit — After installation, the ZeManage ribbon tab appears in the Revit toolbar.
  2. Sign In — Click the ZeManage tab and authenticate with your license credentials. Your device will be registered automatically.
  3. Register Your Model — Open a Revit model and register it via ZeManage > Model Registration. Each model receives a unique GUID for tracking.
  4. Configure Sync Settings — Set your preferred background sync interval, relinquish timeout, and scheduling window.
  5. Explore the Dashboard — Open the Health Monitor to view your model's health score and key metrics.
  6. Set Up Protection (Admins) — Configure pin protection, command restrictions, event rules, and rule-based protection as needed.

Tip: Import a pre-configured .ze file from your administrator to instantly apply sync and protection settings.

Protection

ZeManage provides four layers of model protection to enforce BIM standards and prevent unauthorized changes. Each layer can be configured independently at the company, project, or model level.

Pin Protection

Lock individual elements to prevent accidental modification. Protected (pinned) elements cannot be moved, deleted, or modified without explicit admin authorization. Pin protection is especially useful for critical elements like grids, levels, and reference planes.

  • Pin elements individually or by selection set
  • Visual indicators show which elements are protected
  • Unpinning requires administrator override (OTP or password)

Command Protection

Control which Revit commands can be executed on protected elements. Each command can be set to one of three protection modes:

Protected CommandsAvailable Modes
Delete, Cut, Copy, Move, Rotate, Mirror, Group, Unpin Notify Assist Protect

Notify

Logs the action for audit purposes. The user is informed but allowed to proceed with the operation.

Assist

Shows a guidance dialog explaining the policy. The user can choose to proceed or cancel the action.

Protect

Blocks the action entirely. Requires OTP or admin password to override and proceed.

Event Restriction

Restrict Revit-level events based on company policies. Event restrictions fire automatically when a user attempts a restricted action:

EventDescription
Open from non-approved locationBlock opening files from unauthorized network paths or local drives
Open central file directlyPrevent users from opening central models instead of creating local copies
Model upgrade protectionBlock upgrading models to a newer Revit version without authorization
Save over earlier versionPrevent saving a model that would overwrite an earlier file version
CAD import / explodeBlock importing or exploding CAD files to maintain model purity
Sync conflict detectionDetect and alert on synchronization conflicts with central model
Family loading / version mismatchControl family loading and flag version mismatches
Revit Link pin promptPrompt users to pin Revit links after loading

Rule-Based Protection

Define custom rules based on parameters, categories, or element types. Rules are evaluated in real time as users interact with the model.

ScopeDescriptionManaged By
Company-wideApplied across all projects and models in the organizationCompany Admin
Project-wideApplied to all models within a specific projectProject Admin
Model-specificApplied to a single model onlyProject Admin

Rules sync to the cloud via REST API and are pushed in real time to all connected clients via SignalR. Changes made offline are queued and applied when connectivity is restored.

Admin Override

When a user encounters a protected action, administrators can authorize the operation using:

  • OTP (One-Time Password) — A single-use code generated by an admin, valid for one override action.
  • Admin Password — Direct password entry for immediate override authorization.
  • Evidence Capture — Automatic before/after screenshots are captured for every override to maintain a complete audit trail.

Activity Tracker

Monitor all activity within your Revit environment. The Activity Tracker provides visibility into sessions, crashes, and model changes.

Session Information

View the current Revit session details: active user, machine name, open model, connection status, and session duration.

Crash Register

View crash history with timestamps, automatic Revit journal file analysis, affected model identification, and recovery information.

Model Activities

Track all model changes per session with user attribution, timestamps, element details, and operation type logging.

Journal Analysis: The Crash Register automatically parses Revit journal files to identify the likely cause of crashes, helping teams diagnose and prevent recurring issues.

User Roles

ZeManage uses a role-based access control system with three tiers. Roles determine what features a user can configure and manage.

RoleAccess LevelCapabilities
Company Administrator Full Access All features across all projects. Configure company-wide protection rules, commands, events, and user management. Generate OTPs. View all audit logs.
Project Administrator Project-scoped Manage assigned projects. Configure project/model-level protection, rules, and sync settings. View project-level audit data.
Normal User Standard Work within defined protections. View session info, health dashboard, sync settings, and personal activity. Request overrides via OTP.

Session role-swapping is supported — administrators can temporarily operate as a Normal User to test the end-user experience without changing their actual role assignment.

Evidence Capture

When a protected action is overridden, ZeManage automatically captures screenshots of the affected area before and after the operation. This visual evidence is stored with the audit log entry and synced to the cloud for review by administrators.

OTP Management

One-Time Passwords (OTPs) provide a secure mechanism for administrators to grant temporary override access to protected operations without sharing admin credentials.

  • Generation — Company and Project Administrators can generate OTPs from the admin panel.
  • Single Use — Each OTP is valid for exactly one override action and expires after use.
  • Audit Trail — OTP usage is logged with the override action, including who generated it and who used it.
  • Expiration — OTPs have a configurable expiration window for security.

Troubleshooting & FAQ

Frequently Asked Questions

ZeManage tab is not visible in Revit

Ensure the plugin is installed for the correct Revit version. Check %AppData%\Autodesk\Revit\Addins\ for the ZeManage .addin manifest file. Restart Revit after installation.

Unable to connect to cloud services

Verify your internet connection and check that your organization's firewall allows outbound HTTPS traffic. ZeManage will automatically switch to offline mode and queue operations for later sync.

License activation fails

Confirm that your license key is entered correctly (no leading/trailing spaces). If the key has been used on the maximum number of devices, contact your Company Administrator to deactivate an existing device.

Sync is not running on schedule

Check the Sync Control settings to ensure the schedule window is configured correctly. Background sync requires the Revit model to be open and saved at least once. Verify the sync mode (Continuous vs When Paused) matches your expectations.

Health Monitor shows no data

Run the Analyze Model function from the Health Monitor panel. The initial analysis must be triggered manually; subsequent analyses run according to your configuration.

OTP code is not working

OTP codes are single-use and have an expiration window. Verify the code has not already been used or expired. Request a new OTP from your administrator if needed.

Crash Recovery: If Revit crashes while ZeManage is active, all unsynchronized data is preserved in the local SQLite database. On next launch, the offline queue will automatically process pending operations.
